An AI agent that runs on your machine.

Heya works in your terminal with any model, local or cloud. It reads your code, reproduces bugs, and writes the fix, without sending your project anywhere you didn't choose.

I built Heya for the parts of my job that eat time but don't need my judgment. Reproducing a WooCommerce bug on a clean install, reading a plugin to find the one line that matters, checking a fix before I trust it. It does the legwork in your terminal, with whatever model you point it at, and keeps your code on your machine. I still make the call on what ships.

What you get

Runs on your machine

Point Heya at a local model and nothing leaves your laptop. No account, no telemetry, no project upload. Your code is yours.

# config.toml
[profiles.local]
base_url = "http://localhost:11434/v1"
model = "qwen2.5-coder:14b"

Any model, swapped mid-task

Heya speaks the OpenAI chat API, so any provider works. A local model through Ollama, or Claude, GPT, Gemini, Kimi, GLM, and DeepSeek through OpenRouter. Switch between them mid-conversation.

you › /model cloud-openrouter
switched to cloud-openrouter (claude)

Works from what you have

Drop in a screenshot of the error, a debug log, or any file. With a vision model, Heya reads the screenshot itself, so you skip retyping a stack trace or describing what you see.

you › what is failing here? @checkout-error.png @debug.log
· reading checkout-error.png (image)
· reading debug.log
found a fatal in class-wc-checkout.php:412

Reproduce, diagnose, fix

Heya boots a clean WordPress, reproduces the bug, finds the root cause, and verifies the fix against a real run before it says the word "done".

· wp_playground: clean WP 6.5 / WC 8.7
verified repro passes, regression passes

Brings your own tools

Drop your Claude skills, plugins, and MCP servers in. Heya reads them and uses them, no rewrite. Your setup, in a new agent.

you › /skills
writing-voice, support-reply, wp-security ...

Reviews its own work

Before a change is called done, it runs through a panel: correctness, security, WordPress standards, and over-building. The weak findings get thrown out.

you › review_changes
nothing blocks 1 minor, 0 security

Quickstart

  1. 1

    Install it. The package is heya-agent, the command is heya.

    pipx install heya-agent
  2. 2

    Set up a model. The wizard walks you through local or cloud.

    heya init
  3. 3

    Ask it something real.

    heya "triage this: checkout 500s on WP 6.5 / WC 8.7"

Needs Python 3.11 or newer. A local model needs Ollama; a cloud model needs a key.
